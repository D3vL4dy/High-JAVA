중간 프로젝트 참관 내용

❑ 개요
  - 총 4개조의 발표를 참관
  - 발표 순서 및 구성은 초급 프로젝트와 거의 동일
    (순서 : 프로젝트 소개 → 조원 및 역할 소개 → 설계 구조 및 흐름도 설명 → 프로젝트 시연)


❑ 프로젝트 세부 내용 및 특징
  1. 유기동물 커뮤니티 플랫폼
    ▷ 주제
	: 반려동물에 대한 관심이 늘어가는 추세에 맞춰 반려동물의 보호자끼리 정보를 공유하고 다른 
          보호자나 유기동물들을 입양하는데 도움을 주는 커뮤니티 플랫폼
    ▷ 특징
	① 유기동물에 대한 정보를 API로 받아 보호 장소와 예약을 할 수 있게 구현
	② 동물들을 키우는데 도움이 되는 물품들을 판매 및 결제가 가능하도록 구현
	   (결제 API를 이용해서 네이버페이로 결제가 되는 모습을 시연)
	③ 다이어리 기능을 통해 개인이 키우는 반려 동물에 대한 스토리 및 일기를 남기고 다른 사람과 
	   공유할 수 있도록 구현
	④ 챗봇기능을 메인페이지 좌측 하단에 구현(간단한 질문에 대한 답이 자동으로 출력)
    ▷ 구성
	① 개인 - 관리자 회원으로 구성되어 있으며 각각 웹페이지를 통해 로그인, 로그아웃 가능
	   (로그인 시 이메일 확인, 체크박스를 통해 자동 가입 방지 기능 구현)
	② 공지, 자유 게시판, 다른 사람의 다이어리를 볼 수 있는 게시판으로 구성
	③ 메인 화면 중앙에는 유튜브 링크를 걸어두어 자동으로 주제와 관련된 동영상이 재생
	④ 마이페이지에선 캘린더로 자신의 반려동물 입양 일정을 확인하거나 계획을 추가할 수 있도록 구성
	

  2. 교육생 커뮤니티 시스템
    ▷ 주제
	: 대덕인재개발원 커뮤니티로 수업 일정을 인터넷으로 확인하고 한정된 시간 제약에서 
	  벗어나 강사님들께 질문을 할 수 있으며, 수강생들끼리 자유롭게 소통할 수 있는 커뮤니티
    ▷ 특징
	① 한 페이지에 모든 기능이 보이도록 구성(채팅, 일정, 공지사항 등)
	② 다양한 API나 라이브러리 사용을 통해 편의성 증대
	   (예시 : 카카오API – 회원가입, FUllcalendar – 달력, summernote – 텍스트 관련 에디터, 
	    websocket – 채팅기능)
	③ 게시판 글 작성 시 파일, 그림 등을 첨부할 수 있도록 구현
	④ 1:1채팅, 그룹 채팅방 기능 구현
    ▷ 구성
	① 학생 – 교사 계정으로 구분
	   (로그인 시 이메일 확인, 체크박스를 통해 자동 가입 방지 기능 구현)
	② 다른 창이나 페이지로 넘어가는 구조를 최소화하여 사용자 편의성을 높일 수 있도록 구성
	③ 하나의 글 생성 상자에서 체크박스를 이용해 저장할 게시판을 선택 가능하도록 구성 
  3. 코로나 지원 커뮤니티
    ▷ 주제
	: 엔데믹이 가까워짐과 동시에 코로나 확진, 의심자들은 이제 개인이 스스로 격리하거나 정보를 얻어야 
	  현 상황에 맞춰 서로 정보를 공유하고 도움을 줄 수 있는 커뮤니티
    ▷ 특징
	① 다양한 API를 활용하고 사진이나 그래픽을 통해 표현
	   (예시 : 코로나 확진자 자료를 API로 받고 한반도 모형위에 확진자의 인원수 만큼 표시되는 그래픽,
		  제약회사의 약품에 대한 API를 받아와 개인이 쉽게 약에 대한 정보 제공 및 사진 제공,
		  카카오맵API와 리뷰 작성기능을 통해 진료소에 대한 정보 제공,
		  질병관리청에서 API를 받아와 방역지침과 코로나에 대한 정보 제공)
	② 관리자 로그인과 일반, 전문 사용자의 계정과 구분하였으며 관리자 로그인 페이지로 접속할 수 있는
	   방법을 숨겨 놓음
	③ 오류 상황이 발생하였을 시 알람 기능이 아닌 페이지 내부의 내용이 바뀌면서 더 이상 같은 
	   방법으로 접근할 수 없도록 시스템 구현
	④ 게시글에 신고기능을 추가해서 관리자의 게시판 관리에 도움을 줄 수 있는 기능 구현
    ▷ 구성
	① 일반 – 전문가 – 관리자 계정으로 구분
	   (로그인 시 이메일 확인, 체크박스를 통해 자동 가입 방지 기능 구현)
	② 질의응답, Q&A 게시판에서 글을 클릭했을 때 내용을 보여주는 페이지로 넘어가는 구조가 아닌
	   글을 감싸고 있던 상자가 펼쳐지면서 답변을 볼 수 있게 구성
	   (하나의 상자가 열려있는 상태에서 다른 상자를 클릭하면 이전의 글 상자는 닫히는 구조)
	③ 하나의 글 생성 상자에서 체크박스를 이용해 저장할 게시판을 선택 가능하도록 구성


  4. 영화 예매 사이트
    ▷ 주제
	: 기존 영화관 사이트들의 장점들을 모아 편의성을 증대시킨 예매 사이트 제작 
    ▷ 특징
	① 예매 과정이 종료되면 결제 API기능으로 통해 실제로 결제가 되도록 구현
	② 지도 API를 사용해 위치에 따라 영화관 위치를 표시해주는 기능 구현
	③ 리뷰를 쓸 때 욕설을 감지할 수 있는 필터 구현
	   (욕설 감지 시 알람과 함께 경고글이 텍스트 박스에 올라오도록 구현) 
    ▷ 구성
	① 비회원 – 회원 – 관리자로 구분
	   (로그인 시 이메일 확인, 체크박스를 통해 자동 가입 방지 기능 구현)
	② 비회원으로도 예매 및 결제가 가능하며 예매 후 티켓을 출력할 수 있도록 페이지 구성
	③ 관리자 로그인은 페이지 하단에 체크박스를 클릭해야 로그인이 될 수 있도록 구성
	④ 메인화면의 배경은 상영중인 영화의 포스터로 꾸며져 있으며 일정한 시간 간격을 두고 옆으로 
	   밀려가는 형식을 구현(제어할 수 있는 버튼도 구현)
	⑤ 메이페이지 전면에는 예매율에 따라 영화 포스터가 상단으로 나오며 포스터를 클릭하면
	   해당하는 영화의 예고편을 볼 수 있는 창이 뜨도록 구현
	⑥ 예매 시 남은 좌석을 영화관의 구조를 모형화한 페이지에서 확인할 수 있고 남은 자리를 클릭하면
	   자동으로 해당 좌석의 정보가 입력되는 구조



❑ 후기
   1. 대부분 게시판의 구조는 초급프로젝트 때 진행했던 내용과 흡사했지만 웹페이지의 디자인과 
      페이지구성을 처리하는 방법을 달리하여 개성 있고 조별 선정한 주제에 맞는 사용자UI를 구현하기 위해
      많은 고민과 노력이 있었다는 점을 느낄 수 있었다.
   2. 주제 선정은 자율주제였기에 주로 자신들의 관심분야, 혹은 이미 있는 프로그램이나 웹페이지에서 평소 
       불편함을 느꼈던 부분을 해소하기 위한 나름의 고민이 깊게 들어가 기획 의도는 물론이고 예상 성과까지 
       보여주는 발표도 있었다.
   3. 대략 20일 정도의 시간을 갖고 프로젝트를 진행하며 모든 조가 시간이 부족했었다는 이야기를 하였다.
   4. 다양한 API들을 웹페이지의 디자인에 맞게 표출하는 방법에 따라 완성도의 차이를 보였으며 너무 많은 
      데이터를 한번에 보여주는 경우 종종 페이지가 다운되는 오류가 뜨는 경우가 있었다.
